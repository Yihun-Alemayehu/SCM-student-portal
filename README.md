# Student Portal System — SCM Project

## Project Overview
This repository contains the Software Configuration Management (SCM) project for the **Student Portal System**, developed as an academic mini-project to demonstrate proper configuration control, change management, and release management.

---

## Team Structure & Roles
- **SCM Manager** — Configuration control, baselines, approvals
- **Change Manager** — Change request handling and logging
- **Developers (2)** — Feature implementation
- **QA Testers / Auditors (2)** — Verification, audits, and quality assurance

---

## SCM Artifacts
The project includes the following SCM deliverables:
- Software Configuration Management Plan (SCMP)
- Configuration Item (CI) Register
- Change Request Forms
- Change Log
- Baseline Records (BL1 & BL2)
- Release Notes (v1.0 & v1.1)
- Configuration Audit Report (PCA & FCA)

All official documents are maintained in Word/Excel formats as required for academic submission.

---

## Baselines
- **BL1:** Initial project structure and documentation
- **BL2:** Functional baseline after approved change implementation

Baselines are tagged in Git for traceability.

---

## Change Management
All changes are handled through formal Change Requests (CRs), reviewed and approved before implementation. Each implemented change is traceable through the Change Log and Release Notes.

---

## Repository Structure
/docs → SCM documents (plans, registers, audits)

/src → Application source code

/releases → Release notes
